XAM290 Master-detail and drawer navigation

Exercise 1: Control the Master behavior of a MasterDetailPage (XAM290)

In this exercise, you'll influence how a MasterDetailPage presents the master UI for drawer and master-detail UI.

To complete the exercise, you will need Visual Studio for Windows or macOS with the Xamarin development tools installed. You will also need either an emulator/simulator or a device to run the exercise on. Please see the setup page if you need help installing the Xamarin development environment.

Open the starter solution

Open the starter solution from the Exercise 1 > Start folder in your copy of the cloned or downloaded course materials in either Visual Studio on Windows or Visual Studio for Mac.

Explore the code

This is a minimal application to demonstrate master-detail and drawer navigation in Xamarin.Forms. All of the code you'll be using is in App.xaml.cs, and the UI is defined programmatically.

  1. Open App.xaml.cs.
  2. Find the constructor: notice it's defining both the data and the UI. This application allows you to switch between details for three (3) different colors.
  3. Find the assignment for masterDetailPage.MasterBehavior: notice it's set to MasterBehavior.Popover.
  4. Run the application on any platform/device. Use the menu icon in the top left to open the drawer and update the presented color details page.

Change the master behavior

  1. Change the MasterBehavior assignment to Split and run on a phone device or simulator. Verify that it still uses popover behavior.
  2. Change the MasterBehavior assignment to SplitOnLandscape and SplitOnPortrait and run the application on a tablet or desktop each time. Does it behave as expected?
  3. Run the application on a desktop. Does the behavior change if you resize the window? Do you see the behavior you expect?

Exercise summary

In this exercise, you influenced the presentation of the master page when using a MasterDetailPage. The default behavior may not be exactly as you expected. Understanding how Xamarin.Forms changes behavior based on idiom and orientation is very important when creating applications that run on a large range of device types. Note: Since you're just altering the MasterBehavior, there isn't any Completed solution for this exercise.

You can view the completed solution in the Exercise 1 > Completed folder of your copy of the cloned or downloaded course materials.
